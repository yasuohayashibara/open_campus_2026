# RL LAB — 強化学習ロボット体験

オープンキャンパス向けの、iPad対応・依存ライブラリなしのWebシミュレータです。

## 起動

```bash
python3 telemetry_server.py
```

ブラウザで `http://localhost:8080` を開いてください。 実機走行中の座標・角度・左右モーター出力は画面下部に表示され、`toio-telemetry.jsonl` へ自動保存されます。シミュレータはオフラインで動作します。実機接続にはWeb Bluetooth対応ブラウザとHTTPS配信が必要です。

## 実機用マット

実機で計測したA4マットの対角端（Position ID: 約 `(260,153)`〜`(388,342)`）を使用範囲として設定しています。紙は縦置き状態から時計回りに90°回し、上側だった辺が右側へ来る横長配置にします。画面と実機の左下・右上が一致するよう、位置と角度を90°変換します。

- スタート位置の目安: Position ID `(350, 315)`、画面の右上方向へ向ける（キューブ角度約248°）
- ゴール位置の目安: Position ID `(282, 169)`
- 物理的な紙面と画面を対応させるため、追加の安全余白は設けない
- 実走行に使用する生座標範囲: X `260–388` / Y `153–342`
- ゴール到達時は緑色で2秒、座標で確定した場外停止時は赤色で0.8秒点灯し、ブラウザにも大きな赤い警告を表示（センサー異常時はLED消灯）
- 実機は障害物を直接センシングしないため、学習画面に表示される既知座標へ実物を置く（コースA: O1 `(291,285)`, O2 `(346,246)`, O3 `(302,206)`, O4 `(356,193)`）

キューブを持ち上げると即停止し、マットへ戻してPosition IDを1回取得すると学習済み方策をすぐ自動再開します。

大型プレイマットへ切り替える場合は、`app.js` の `MAT_PROFILE` を `MAT_PROFILES.large` に変更してください。マットを見失った場合とコース端へ達した場合は、安全のためモーターを停止します。 接続後の「AI方策を再送信」ボタンは、デバイスを再選択せずに同じ学習済み方策を停止・再開します。

## 体験フロー

1. 報酬パラメータを調整してAIの性格を設計
2. 15秒間、96体を並列評価する実学習をリアルタイム表示
3. 学習済みモデルをtoioへ転送するSim-to-Realデモ
## 実学習モデル

依存ライブラリなしの進化戦略で `5入力 → 8中間 → 2モーター出力` のニューラルネットワークを学習します。96体を並列シミュレーションし、ゴールへの接近・到達、障害物への接近・衝突、移動速度から報酬を計算します。画面の試行回数、成功率、報酬、ネットワーク重みはすべて実際の評価結果です。学習完了後は「＋10秒追加学習」を何度でも実行でき、個体群と最高重みを保持したまま世代を継続します。転送時はゴールへ到達した個体を優先し、衝突個体は選びません。学習完了画面では転送対象個体の軌跡を緑色で表示します。成功個体がない場合は転送ボタンを表示しないため、追加学習してください。実機では、Position IDから学習時と同じ5入力を毎回計算し、学習済みネットワークの2出力を左右モーターへ直接送ります。ゴール方向を向かせる別のルール制御は混ぜず、後退やその場旋回を含む学習結果を閉ループで再現します。実機速度だけはA4上で安全に確認できる最大34へ換算し、学習シミュレーションと同等の加速度制限を50ミリ秒周期で適用して滑らかに変化させます。
### パラメータのトレードオフ

探索の幅は初期重みの分散と世代ごとの突然変異量へ、接近報酬はゴールへ近づいた距離の報酬倍率へ、ゴール報酬は到達時の報酬へ、衝突ペナルティは衝突および制動不足の罰へ直接反映されます。探索が小さすぎると局所解に、大きすぎると未収束になります。接近報酬だけが高いと途中で満足し、ゴール報酬が低いと完走する理由が弱くなります。衝突罰が高すぎると動かない方策が有利になるため、4項目のバランスが必要です。
