# RL LAB — 強化学習ロボット体験

オープンキャンパス向けの、iPad対応・依存ライブラリなしのWebシミュレータです。

## 起動

```bash
python3 telemetry_server.py
```

ブラウザで `http://localhost:8080` を開いてください。 実機走行中の座標・角度・左右モーター出力は画面下部に表示され、`toio-telemetry.jsonl` へ自動保存されます。接続後は同じボタンが「toioを切断」に変わり、安全停止して切断できます。シミュレータはオフラインで動作します。実機接続にはWeb Bluetooth対応ブラウザとHTTPS配信が必要です。

## 実機用マット

12枚組A3開発者向けマットの連番面（裏面）に対応しています。キューブから届くPosition IDによりシート #01〜#12を自動判定し、各シートの左上をローカル原点として同じコースへ正規化します。どの番号のシートでもコードの設定変更は不要です。シートは番号が正立する横置きで使用してください。

- 検出範囲: X `34–949` / Y `35–898`（3列×4段、各座標からシートを自動判定）
- シート内の運用範囲: 生座標の各端から約15内側（A3用紙では四辺から約20 mm）
- スタート位置の目安（シート内）: 画面左下付近
- ゴール位置の目安（シート内）: 画面右上付近
- ロボット本体が用紙からはみ出しにくいよう、A3の四辺に約20 mmの余白を設ける
- ゴール到達時は緑色で2秒、座標で確定した場外停止時は赤色で0.8秒点灯し、ブラウザにも大きな赤い警告を表示（センサー異常時はLED消灯）
- 実機コースは `R01` に固定し、障害物3個とゴールを全参加者で同じ位置に置く
- A3原寸配置図: [`print/real-course-a3.pdf`](print/real-course-a3.pdf)（編集・ブラウザー印刷用SVGも同梱）
- Position ID座標値はミリメートルではないため、印刷後にコース外枠が380 × 257 mmであることを確認する

キューブを持ち上げると即停止し、マットへ戻してPosition IDを1回取得すると学習済み方策をすぐ自動再開します。

旧A4マットまたは大型プレイマットへ切り替える場合は、`app.js` の `MAT_PROFILE` をそれぞれ `MAT_PROFILES.a4` または `MAT_PROFILES.large` に変更してください。マットを見失った場合とコース端へ達した場合は、安全のためモーターを停止します。

## 体験フロー

1. 開始画面で、ロボットの動きが異なる初級・中級・上級を選択
2. すべての報酬が「無」の状態を8秒間観察し、学習が進まないことを確認
3. 運用時間に合わせ、4つの報酬を「無・弱・中・強」から選んでAIの性格を設計
4. 12種類から学習環境を選び、10秒間・1,200ステップ、96体を並列評価する実学習をリアルタイム表示
5. 未学習の固定12環境を開始角度10通り、合計120試行で評価
6. 歴代ベストモデルをtoioへ接続し、ステージへ置いて走行させるSim-to-Realデモ

開始画面では「チュートリアルから始める」と「チュートリアルをスキップ」を選べます。スキップは確認ダイアログを1回表示してから「報酬と環境」へ進みます。

iPad横画面では、4段階それぞれの主操作が1画面に収まるよう情報を絞ります。「報酬と環境」では報酬を左へ寄せ、右側に12環境を4列×3行ですべて表示します。性格診断は報酬欄の一文へ集約します。学習画面では環境選択を行わず、マップ、残り時間、試行数、成功率、衝突数を優先します。評価結果は「今回の候補」と「歴代ベスト」を並べ、その直下に「同じ条件で追加学習」「報酬と環境を選び直す」「現実への転送へ進む」の3操作を配置します。各ボタンには、ベストAIから再開すること、ネットワークを保持すること、次画面で接続することを補足表示します。試行ログは下へスクロールすると確認できます。報酬内訳と自動アドバイスは通常表示から外し、実機の座標・モーター情報は常時表示の「運用ログ」にまとめます。
## 実学習モデル

依存ライブラリなしの進化戦略で `7入力 → 8中間 → 2モーター出力` のニューラルネットワークを学習します。ゴールと障害物の情報に加えて現在の前進速度・旋回速度を観測し、96体を並列シミュレーションします。走行難易度が上がるほど加減速、角加速度、高速時の旋回限界が厳しくなり、カーブ手前の減速が必要です。ゴールへの接近・到達、障害物への接近・衝突、移動速度から報酬を計算します。画面の試行回数、成功率、報酬、ネットワーク重みはすべて実際の評価結果です。学習完了後は「＋10秒追加学習」を何度でも実行でき、個体群と最高重みを保持したまま世代を継続します。訓練後は成功個体を優先して固定12環境で評価し、共通の実機適性が最も高い歴代ベストを転送します。合否は比較と改善の目安として表示し、基準未達でも評価済みの歴代ベストを実機で観察できます。実機ではBluetooth接続後、ステージ上のPosition IDを検出すると自動走行し、持ち上げると停止します。ステージへ置き直すと自動的に走行を再開します。Position IDに現在のモーター状態を加えた学習時と同じ7入力を毎回計算し、学習済みネットワークの2出力を左右モーターへ送ります。実機速度はマット上で安全に確認できる最大34へ換算し、選択した難易度に応じた加速度制限を50ミリ秒周期で適用して滑らかに変化させます。
### パラメータのトレードオフ

4つの項目は細かなスライダーではなく「無・弱・中・強」の4段階で設定します。探索の幅は初期重みの分散と世代ごとの突然変異量へ、接近報酬はゴールへ近づいた距離の報酬倍率へ、ゴール報酬は到達時の報酬へ、衝突ペナルティは衝突および制動不足の罰へ直接反映されます。探索が小さすぎると局所解に、大きすぎると未収束になります。接近報酬だけが高いと途中で満足し、ゴール報酬が低いと完走する理由が弱くなります。衝突罰が高すぎると動かない方策が有利になるため、4項目のバランスが必要です。

### 複数環境での追加学習

追加学習では、固定した12種類の学習環境から次の配置を選びます。96個体の重みと世代数はリセットするまで保持されますが、新環境の成功率・衝突数・最高報酬はその環境で再評価します。訓練終了後は毎回、訓練には使わない固定12環境（障害物2個・4個・6個の環境を各4）を、開始角度10通りずつ合計120試行で同時評価します。開始位置と配置は固定し、角度は約−26度から＋26度までの固定セットを全候補へ共通に使います。評価中は重み更新、突然変異、個体選択を行いません。成功、衝突、時間、安全距離から共通の実機適性を計算し、70走以上の成功かつ衝突40走以下を実機候補とします。画面では今回と直前までの歴代ベストについて、選んだ4段階の報酬設定、走行難易度、学習環境ID、累計学習時間、120試行の成功数・衝突数・評価報酬を比較します。

120試行の評価結果は、環境ごとの成功数、報酬4項目、走行難易度、直近の学習環境ID、累計学習時間、世代数、総成功数、総衝突数、評価報酬とともにページ内メモリーへ最大50件記録します。画面には直近12件を表示し、評価基準を優先したうえで実機適性が最も高い行を `BEST` として示します。学習済みネットワークを明示的にリセットした場合は、試行番号を増やさず `RESET` 行を残します。「画面のログだけ消去」は表示中のログだけを消し、サーバーの `experiment-results.jsonl` と学習済みネットワークには影響しません。ブラウザーの再読み込みやページ離脱では状態が失われるため、体験開始後は確認警告を表示します。実機画面から「報酬と環境」または「学習結果」へ戻る際は、toioを停止・切断し、学習済みネットワークは保持します。全状態の消去は画面操作としては設けず、必要な場合だけページを再読み込みします。

追加学習では、現在の実験内で固定12環境を通過したベストだけをチェックポイントとして個体群へ戻します。ベスト個体そのものを保護しつつ、その周囲へ31候補を生成して新環境を探索します。評価結果が実験内ベストを下回った更新は「見送り」としてログへ残し、次の追加学習も検証済みベストから再開します。評価結果から報酬設計画面へ戻ってもネットワーク、実験内ベスト、学習履歴を保持し、変更後の報酬と環境を使って追加学習します。「学習ネットワークだけリセット」は報酬・環境画面に置き、確認後に現在の個体群と実験内ベストだけを消して新しく学習します。全体の歴代ベストと試行ログは消さず、比較表示と実機転送に引き続き使います。

学習の進行は `requestAnimationFrame` の呼び出し回数ではなく実経過時間から計算します。初回と追加学習はどちらも10秒・毎秒120ステップで、1回あたり1,200ステップ（4世代）です。60Hz・120Hz・144Hzの画面で学習速度は変わりません。固定12環境の表示評価も毎秒60ステップに固定します。

### 12種類の学習環境

報酬設計画面では、障害物の位置と通路の狭さが異なる `E01`〜`E12` から学習環境を選びます。「やさしい」「ほどよい」「むずかしい」の3段階があり、簡単すぎる環境では回避を学ぶ機会が少なく、難しすぎる環境では成功例を得にくくなります。評価後は同じ12環境から次の追加学習先を選び、同じ環境を反復した場合と難易度・配置を変えた場合を比較できます。本番テストには、これらとは別の未学習12環境を使います。

### 報酬と本番成果の見方

報酬はAIが良い行動を探すための「学習用の点数」であり、最終成績ではありません。評価画面では、未学習の12環境におけるゴール数と衝突数を本番テストの成果として優先表示します。直前までの歴代ベストよりAIの点数が80以上増えてもゴール数が増えない場合は「報酬のワナ」と判定し、高得点の取り方と目的達成が一致していないことを説明します。点数とゴールが両方増えた場合、点数が下がってもゴールが増えた場合にも、それぞれ異なる学習ポイントを表示します。ページ内の試行ログでは直前の試行と同じ判定を行い、比較の経過を残します。
