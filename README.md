# RL LAB — 強化学習ロボット体験

オープンキャンパス向けの、iPad対応・依存ライブラリなしのWebシミュレータです。

## 起動

```bash
python3 telemetry_server.py
```

ブラウザで `http://localhost:8080` を開いてください。 実機走行中の座標・角度・左右モーター出力は画面下部に表示され、`toio-telemetry.jsonl` へ自動保存されます。シミュレータはオフラインで動作します。実機接続にはWeb Bluetooth対応ブラウザとHTTPS配信が必要です。

## 実機用マット

12枚組A3開発者向けマットの連番面（裏面）に対応しています。キューブから届くPosition IDによりシート #01〜#12を自動判定し、各シートの左上をローカル原点として同じコースへ正規化します。どの番号のシートでもコードの設定変更は不要です。シートは番号が正立する横置きで使用してください。

- 検出範囲: X `34–949` / Y `35–898`（3列×4段、各座標からシートを自動判定）
- シート内の有効範囲: X 約 `0–305` / Y `0–215`
- スタート位置の目安（シート内）: 画面左下付近
- ゴール位置の目安（シート内）: 画面右上付近
- 物理的な紙面と画面を対応させるため、追加の安全余白は設けない
- ゴール到達時は緑色で2秒、座標で確定した場外停止時は赤色で0.8秒点灯し、ブラウザにも大きな赤い警告を表示（センサー異常時はLED消灯）
- 実機は障害物を直接センシングしないため、学習完了画面に表示される現在のシート用Position IDへ実物を置く

キューブを持ち上げると即停止し、マットへ戻してPosition IDを1回取得すると学習済み方策をすぐ自動再開します。

旧A4マットまたは大型プレイマットへ切り替える場合は、`app.js` の `MAT_PROFILE` をそれぞれ `MAT_PROFILES.a4` または `MAT_PROFILES.large` に変更してください。マットを見失った場合とコース端へ達した場合は、安全のためモーターを停止します。 接続後の「AI方策を再送信」ボタンは、デバイスを再選択せずに同じ学習済み方策を停止・再開します。

## 体験フロー

1. 開始画面で、ロボットの動きが異なる初級・中級・上級を選択
2. すべての報酬が「無」の状態を8秒間観察し、学習が進まないことを確認
3. 10分の体験時間内で、4つの報酬を「無・弱・中・強」から選んでAIの性格を設計
4. 15秒間、96体を並列評価する実学習をリアルタイム表示
5. 未学習の固定12環境で汎化性能と報酬内訳を自動評価
6. 歴代ベストモデルをtoioへ転送するSim-to-Realデモ
## 実学習モデル

依存ライブラリなしの進化戦略で `7入力 → 8中間 → 2モーター出力` のニューラルネットワークを学習します。ゴールと障害物の情報に加えて現在の前進速度・旋回速度を観測し、96体を並列シミュレーションします。走行難易度が上がるほど加減速、角加速度、高速時の旋回限界が厳しくなり、カーブ手前の減速が必要です。ゴールへの接近・到達、障害物への接近・衝突、移動速度から報酬を計算します。画面の試行回数、成功率、報酬、ネットワーク重みはすべて実際の評価結果です。学習完了後は「＋10秒追加学習」を何度でも実行でき、個体群と最高重みを保持したまま世代を継続します。訓練後は成功個体を優先して固定12環境で評価し、共通の実機適性が最も高い歴代ベストだけを転送します。合格モデルがない場合は転送ボタンを表示しないため、報酬を見直すか追加学習してください。実機では、Position IDに現在のモーター状態を加えた学習時と同じ7入力を毎回計算し、学習済みネットワークの2出力を左右モーターへ送ります。実機速度はマット上で安全に確認できる最大34へ換算し、選択した難易度に応じた加速度制限を50ミリ秒周期で適用して滑らかに変化させます。
### パラメータのトレードオフ

4つの項目は細かなスライダーではなく「無・弱・中・強」の4段階で設定します。探索の幅は初期重みの分散と世代ごとの突然変異量へ、接近報酬はゴールへ近づいた距離の報酬倍率へ、ゴール報酬は到達時の報酬へ、衝突ペナルティは衝突および制動不足の罰へ直接反映されます。探索が小さすぎると局所解に、大きすぎると未収束になります。接近報酬だけが高いと途中で満足し、ゴール報酬が低いと完走する理由が弱くなります。衝突罰が高すぎると動かない方策が有利になるため、4項目のバランスが必要です。

### 複数環境での追加学習

追加学習では、障害物の個数を「2個・4個・6個」から選び、障害物配置を更新します。個数の効果を比較しやすくするため、障害物が増える環境でも大きさと配置の揺れは急増させません。96個体の重みと世代数はリセットするまで保持されますが、新環境の成功率・衝突数・最高報酬はその環境で再評価します。訓練終了後は毎回、訓練には使わない固定12環境（障害物2個・4個・6個の環境を各4）を縮小フィールドで同時表示して候補方策を自動評価し、重み更新、突然変異、個体選択は行いません。成功、衝突、時間、安全距離から共通の実機適性を計算し、歴代ベストを更新します。画面では今回と直前までの歴代ベストについて、選んだ4段階の報酬設定、走行難易度と障害物数、累計学習時間、12環境の成功数・衝突数・評価報酬を比較します。さらに、成功数、衝突数、学習時間、訓練環境、報酬設定を組み合わせて、次に1つだけ変える項目を助言します。詳しい報酬内訳は必要なときだけ展開できます。評価画面のボタンからすぐ報酬設計へ戻り、パラメータを変えて再学習できます。7走以上の成功かつ衝突4走以下の歴代ベストだけが実機転送対象となり、練習環境固有の軌跡誘導は含めません。

固定12環境の評価結果は、報酬4項目、走行難易度、学習した障害物数の順序、累計学習時間、世代数、成功数、衝突数、評価報酬とともにページ内メモリーへ最大50件記録します。画面には直近12件を表示し、実機候補の判定を優先したうえで実機適性が最も高い行を `BEST` として示します。ページを再読み込みすると画面ログとモデル状態を初期化します。調整検証用の同じ評価レコードはサーバーの `experiment-results.jsonl` に保存します。

追加学習では、現在の実験内で固定12環境を通過したベストだけをチェックポイントとして個体群へ戻します。ベスト個体そのものを保護しつつ、その周囲へ31候補を生成して新環境を探索します。評価結果が実験内ベストを下回った更新は「見送り」としてログへ残し、次の追加学習も検証済みベストから再開します。報酬設計画面から開始する新しい実験では、個体群、実験内ベスト、学習環境履歴を必ず初期化します。全実験の歴代ベストは比較表示と実機転送だけに使い、新しい実験の学習へは混ぜません。

学習の進行は `requestAnimationFrame` の呼び出し回数ではなく実経過時間から計算します。調整可能な内部パラメータとして、初回は毎秒80ステップ、追加学習は毎秒360ステップに分離します。15秒の初回は1,200ステップ（4世代）、追加10秒は3,600ステップ（12世代）です。60Hz・120Hz・144Hzの画面で学習量は変わりません。固定12環境の表示評価も毎秒60ステップに固定します。
